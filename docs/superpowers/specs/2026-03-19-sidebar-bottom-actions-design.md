# Sidebar Bottom Actions Design

**Date:** 2026-03-19

## Background

当前 Flutter 侧边栏同时存在两套导航表达：
- 上部静态菜单包含 `我的`、`更新`、`设置`
- 底部固定区又额外提供 `我的应用`、`下载管理`、`设置` 三个并排图标

这会造成入口重复，也让底部区域和主菜单在视觉语言上不一致。

## Confirmed Requirements

- 顶部菜单顺序调整为：`推荐` → `全部` → `排行` → 服务端动态菜单
- 底部固定区保留三个入口：`我的应用` → `下载管理` → `设置`
- 底部入口改成与上部菜单一致的纵向菜单样式，不再使用横向并排图标
- 继续保留当前按窗口宽度自动折叠的行为，不增加手动折叠开关
- 侧边栏实现需要保持高 UI 性能和高响应速度

## Design

### Navigation Structure

- `SidebarMenuItem` 只保留顶部静态导航：`推荐 / 全部 / 排行`
- 服务端动态菜单继续插入在顶部静态导航之后
- 底部固定区单独维护三个动作项：`我的应用 / 下载管理 / 设置`
- `我的应用`、`设置` 走路由高亮态；`下载管理` 维持打开弹窗的动作语义

### Performance Constraints

- 保持“顶部滚动区 + 底部固定区”的简单布局，避免额外嵌套和重复状态源
- 移除侧边栏对更新红点 Provider 的订阅，减少与本次导航无关的重建
- 继续复用现有 200ms 宽度动画和轻量 hover 状态，不引入新的全局状态

### Testing

- 新增 widget 测试覆盖：顶部菜单顺序、动态菜单位置、底部固定菜单顺序
- 回归 `download_manager_dialog` 相关测试，确认底部入口改成文字菜单后仍可正常打开/关闭弹窗
