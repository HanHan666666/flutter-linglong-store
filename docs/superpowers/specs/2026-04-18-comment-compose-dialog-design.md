# 应用详情评论弹窗设计

日期：2026-04-18

## 背景

当前应用详情页评论区在页面内直接展示评论输入框、版本胶囊和提交按钮。用户希望把“发评论”功能移到弹窗里，避免详情页正文中长期占用一块输入区域。

## 目标

- 评论列表继续留在详情页正文
- 评论输入区改为按钮触发的弹窗
- 提交流程继续复用现有 `appDetailProvider.submitComment()`
- 提交成功后继续回源刷新最新评论

## 选型

采用最小改动方案：

- `AppDetailCommentSection` 页面内保留标题、错误态、空态、评论列表
- 当 `canSubmitComment=true` 时，只显示一个“发表评论”入口按钮
- 点击按钮后打开评论弹窗，弹窗承载原有输入框、版本胶囊和提交按钮

## 非目标

- 不改评论列表接口和提交接口
- 不新增登录、评分、点赞等能力
- 不调整评论列表在详情页中的位置

## 关键细节

- 版本选择胶囊和“展开全部/收起”逻辑整体搬进弹窗
- 弹窗提交成功后关闭；失败时保留弹窗并继续显示现有错误反馈
- 未安装时仍然不显示评论入口，但继续保留评论列表只读展示
