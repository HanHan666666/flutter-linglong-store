## 一、已完成项（14项）

### 1. 详情页复制应用 ID
- **需求来源**：2.4 应用详情与版本操作
- **描述**：应用详情页必须支持复制应用 ID（目前只有进程右键菜单有此功能）
- **状态**：✅ 已完成
- **实现思路**：在详情页头部 appId 文字旁添加点击复制按钮，调用 `Clipboard.setData()`

### 2. 降级确认（安装低版本覆盖高版本）
- **需求来源**：2.4 应用详情与版本操作
- **描述**：当用户安装的版本低于已安装版本时，必须给出降级确认弹窗
- **状态**：✅ 已完成
- **实现思路**：在版本列表安装操作时，对比目标版本与已安装版本，若低则弹出 `ConfirmDialog` 确认降级

### 3. 强制重装确认（安装已存在版本）
- **需求来源**：2.4 应用详情与版本操作
- **描述**：当用户安装的版本与已安装版本完全一致时，必须给出强制重装确认
- **状态**：✅ 已完成
- **实现思路**：安装操作前判断 `installedApps` 中是否存在完全匹配的 `appId + version + arch`，若存在则弹出重装确认

### 4. 卸载运行中应用确认
- **需求来源**：2.4 / 2.5
- **描述**：当卸载应用时，若该应用当前正在运行中，必须给出"强制关闭并卸载"的确认
- **状态**：✅ 已完成
- **实现思路**：卸载前检查 `runningProcessProvider` 中是否存在该 `appId`，若存在则弹出确认弹窗，确认后先 `killProcess` 再 `uninstall`

### 5. 意见反馈（含上传日志文件）
- **需求来源**：2.8 设置、关于与反馈
- **描述**：关于页必须提供意见反馈入口，反馈支持可选上传本地日志文件
- **状态**：✅ 已完成
- **实现思路**：
  1. 关于区域增加"意见反馈"按钮，点击弹出反馈对话框
  2. 对话框含标题、内容文本框、可选日志文件选择器
  3. 先调用 `/app/uploadLog` 可选上传日志，再调用 `/visit/suggest` 提交反馈内容

### 6. 匿名统计上报（启动访问/安装卸载统计）
- **需求来源**：4. 数据与集成需求
- **描述**：应用必须记录匿名启动访问数据和安装/卸载统计数据
- **状态**：✅ 已完成
- **实现思路**：
  1. 实现 `AnalyticsRepositoryImpl`（调用 `linglong-server` 统计接口）
  2. 在 `launch_provider.dart` 启动完成后上报启动事件
  3. 在 `install_queue_provider.dart` 安装/卸载成功后上报事件

- **需求来源**：2.2 全局框架与导航
- **描述**：侧边栏必须渲染服务端通过 `getSidebarConfig()` 下发的自定义菜单项，点击进入对应专题页
- **状态**：✅ 已完成
- **实现思路**：在 `Sidebar` 组件中监听 `sidebarConfigProvider`，动态渲染服务端菜单项，点击路由至 `custom_category_page`

### 8. 设置页四个特定开关
- **需求来源**：2.8 设置
- **描述**：
  - "启动时检查商店版本更新"开关
  - "容器内自动更新商店本体"开关
  - "是否显示基础运行服务"开关（已安装应用列表中过滤基础运行时）
  - "清理废弃基础服务"操作按钮
- **状态**：✅ 已完成
- **实现思路**：
  1. 在 `setting_provider.dart` / `preferences_service.dart` 中增加对应偏好项
  2. 在设置页 UI 中添加 `SwitchListTile` 和操作按钮
  3. "清理基础服务"调用 `ll-cli` `prune` 命令

### 9. 关于页完善
- **需求来源**：2.8 关于
- **描述**：关于页缺少：已收录应用总数量、检查商店新版本按钮、意见反馈入口
- **状态**：✅ 已完成
- **实现思路**：
  1. 已收录数量：调用 `/app/count` 或 `/app/list?page=1&size=1` 接口取 `total` 字段

  3. 意见反馈入口：见第5项

### 10. 安装后直接打开应用
- **需求来源**：2.6 下载管理
- **描述**：安装成功后，下载管理弹窗中的历史记录应提供"打开"按钮
- **状态**：✅ 已完成
- **实现思路**：在 `install_queue_provider.dart` 任务完成回调中，若 `autoRunAfterInstall == true` 则调用 `runApp()`；下载管理弹窗历史项增加"打开"按钮

### 11. 下载管理实时网络速度
- **需求来源**：2.6 下载管理
- **描述**：下载管理必须显示实时网络速度（KB/s 或 MB/s）
- **状态**：✅ 已完成
- **实现思路**：
  1. 新增 `network_speed_provider.dart`，按秒读取 `/proc/net/dev` 计算下载速度
  2. 安装按钮与下载管理弹窗复用同一速度来源，统一展示 `KB/s` / `MB/s`
  3. 当前任务与安装中的更新入口均显示实时下载速度

### 12. 分类页分类栏展开/折叠
- **需求来源**：2.3 应用发现与浏览
- **描述**：分类页的分类栏应支持展开/折叠，默认折叠只显示部分分类
- **状态**：✅ 已完成
- **实现思路**：在 `AllAppsPage` 的 `CategoryFilterHeader` 中增加展开/折叠状态；折叠态使用顶部单行横向分类条，展开态直接把同一顶部容器切换为多行分类区，不再额外渲染下方第二块重复分类面板

### 13. 自定义专题页排序/过滤规则透传
