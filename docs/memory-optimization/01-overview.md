# 01 - 总览与目标

## 1.1 背景

玲珑商店 Flutter 版当前运行内存占用约 **700MB**，远超同类桌面应用的合理水平（300~500MB）。对于一个商店类应用，这一数值偏高，会影响低配设备的可用性和用户体验。

## 1.2 优化目标

| 指标 | 当前值 | 目标值 |
|------|--------|--------|
| 稳态内存（RSS） | ~700 MB | ≤ 500 MB |
| 启动后 30s 静置内存 | ~500 MB（推测） | ≤ 350 MB |
| 浏览 10 个详情页后峰值 | ~800+ MB（推测） | ≤ 550 MB |

## 1.3 优化原则

1. **效果优先**：优先处理投入产出比最高的优化项
2. **安全可控**：每个优化项独立可回滚，不引入新 bug
3. **逻辑等价**：不改变业务行为，不影响用户体验流畅度
4. **可度量**：每项优化前后都有量化对比数据

## 1.4 优化项优先级总表

| 优先级 | 优化项 | 预估节省 | 改动面 | 风险 |
|--------|--------|----------|--------|------|
| 🔴 P0 | 图片 ImageCache 限额生效 | 40~60 MB | 极小（1 行） | 极低 |
| 🔴 P0 | Image.network 3 处加 cacheWidth/cacheHeight | 30~80 MB | 小（3 处） | 低 |
| 🔴 P0 | 截图预览页全尺寸改为 ResizeImage | 10~30 MB/张 | 小（1 处） | 低 |
| 🟡 P1 | 4 个页面补 VisibilityAwareMixin | 30~60 MB | 中（4 文件） | 中 |
| 🟡 P1 | 搜索列表页移除 wantKeepAlive | 5~15 MB | 极小 | 低 |
| 🟡 P1 | 排行榜 pageSize 100→30+分页 | 10~20 MB | 中 | 中 |
| 🟢 P2 | Hive cache Box → LazyBox + 过期清理 | 10~30 MB | 中 | 中 |
| 🟢 P2 | installedAppsProvider 瘦身 | 5~15 MB | 小 | 低 |
| 🟢 P2 | 内存监控工具集成 | 0（辅助） | 小 | 无 |

## 1.5 预期时间线

| 阶段 | 内容 | 完成标志 |
|------|------|----------|
| Phase 1 | P0 图片内存优化（3 项） | 静置内存降至 ~550MB |
| Phase 2 | P1 KeepAlive + Provider 优化 | 稳态内存降至 ~480MB |
| Phase 3 | P2 Hive + 监控 + 收尾 | 稳态内存 ≤ 450MB，峰值 ≤ 500MB |
