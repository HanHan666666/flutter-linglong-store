# 下载中心任务工作面板设计

**日期**：2026-06-08  
**状态**：用户已确认方案 2  
**范围**：侧边栏触发的下载管理弹窗 UI

## 背景

当前下载中心功能链路完整，但视觉表达偏粗糙：弹窗宽度偏窄，标题区、概览区、当前任务、队列与历史记录之间的层级不够稳定，底部实时速度信息也像附加补丁。用户希望在保持现有系统 UI 风格的前提下，让下载中心更精致，并明确提出可以参考图片预览灯箱弹窗的布局。

本设计只重做下载中心的视觉组织与布局，不改变安装队列状态机、不新增 `ll-cli` 调用、不改变复制日志、取消、重试、打开和移除等业务行为。

## 目标

- 将下载中心从“普通卡片弹窗”调整为更像桌面端轻工作面板的结构。
- 参考图片预览灯箱的稳定容器、48px 顶栏、主体舞台和底部辅助栏。
- 明确区分当前任务、等待队列与历史记录，保证当前任务拥有最高视觉优先级。
- 保持现有主题令牌、字体层级、无障碍语义和 Tooltip 完整文本约定。
- 继续复用 `installQueueProvider`、`networkSpeedProvider` 和 `InstallTask.commandOutput`，不新增业务数据源。

## 非目标

- 不改下载中心入口位置和触发方式。
- 不增加暂停、继续、批量操作、优先级调整等新功能。
- 不改严格串行安装规则。
- 不将下载中心升级为独立页面。
- 不用硬编码截断替代 `TextOverflow.ellipsis`，Tooltip 和复制内容必须保留完整源文本。

## 方案

采用“图片预览灯箱风格的任务工作面板”：

1. 外层弹窗使用更宽的稳定面板，宽度约 `640px`，高度在桌面窗口内自适应限制。容器圆角收敛到 `12px`，阴影沿用 modal 阴影。
2. 顶栏高度固定为 `48px`，左侧使用下载图标与标题，右侧放清空历史和关闭按钮。关闭按钮参考图片预览灯箱的小型 hover 关闭控件。
3. 顶栏下方放概览条，展示进行中、等待中、已完成三个指标。该区域是任务状态总览，不承载复杂操作。
4. 主体区域只让任务列表滚动。当前任务使用高亮主卡展示应用图标、状态、进度、百分比、速度和阶段文案；队列与历史记录使用更紧凑的统一行。
5. 底部状态栏固定展示实时速度、任务记录数和轻量提示，承担辅助信息职责，不抢当前任务视觉焦点。

## 约束

- 文件修改优先收敛在 `lib/presentation/widgets/download_manager_dialog.dart`。
- 任务卡片中的业务动作继续通过现有回调进入 `installQueueProvider` 和 `linglongCliRepositoryProvider`。
- `InstallTask.commandOutput` 为空时隐藏“复制日志”，不能回退复制状态文案。
- 当前任务速度优先使用 `task.cliSpeed`，再回退到 `networkSpeedProvider.formatted`。
- 错误文案仍支持完整显示，状态文案 Tooltip 必须使用 `InstallTask.displayMessage` 还原后的完整文本。
- 新增或修改代码注释必须使用中文，并说明设计原因、边界或业务约束。

## 测试策略

- 更新下载中心 widget 测试，覆盖新工作面板关键结构：
  - 弹窗宽度不再停留在旧的窄面板；
  - 顶栏、概览条、任务列表和底部状态栏存在；
  - 当前任务显示“当前任务”高优先级标签；
  - 进度百分比、速度和 `LinearProgressIndicator` 仍按归一化值展示。
- 保留既有行为测试：
  - 关闭弹窗后队列刷新不报错；
  - 失败长文案不被行数截断；
  - 复制日志只复制 `commandOutput`；
  - 失败态状态标签与操作按钮垂直对齐。

## 验收标准

- 下载中心视觉上接近图片预览灯箱的面板语言：稳定顶栏、主体区域、底部辅助栏。
- 当前任务明显比等待和历史任务更突出。
- 队列和历史任务更紧凑，操作区排列清晰。
- 现有下载中心行为和无障碍语义不回退。
- 定向 widget 测试通过，`flutter analyze` 不引入新的 warning 或 error。
