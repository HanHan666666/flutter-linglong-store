## 通用网格布局

几乎所有应用列表页使用同一套网格系统：

```css
/* 原始 CSS */
grid-template-columns: repeat(auto-fill, minmax(18rem, 1fr));
gap: 16px; /* 1rem */
```

Flutter 等效：
```dart
SliverGrid(
  gridDelegate: SliverGridDelegateWithMaxCrossAxisExtent(
    maxCrossAxisExtent: 320, // ~18rem + 弹性
    mainAxisExtent: 120,     // 卡片高度
    crossAxisSpacing: 16,
    mainAxisSpacing: 16,
  ),
)
```

分页参数：`pageSize = 60`（除排行榜页为 100）

## 一、推荐页（RecommendPage）

路由：`/`

### 1.1 结构

```
┌──────────────────────────────────────────┐
│  ┌── 轮播区域 (ApplicationCarousel) ──┐  │
│  │  Banner 1 → Banner 2 → Banner 3    │  │
│  └────────────────────────────────────┘  │
│                                          │
│  ┌── 筛选/排序栏 ─────────────────────┐  │
│  │  [全部|办公|...]  排序:[默认|下载量] │  │
│  └────────────────────────────────────┘  │
│                                          │
│  ┌─┐ ┌─┐ ┌─┐ ┌─┐                        │
│  │ │ │ │ │ │ │ │  ← ApplicationCard 网格 │
│  └─┘ └─┘ └─┘ └─┘                        │
│  ┌─┐ ┌─┐ ┌─┐ ┌─┐                        │
│  │ │ │ │ │ │ │ │                         │
│  └─┘ └─┘ └─┘ └─┘                        │
│                                          │
│  加载中... / 没有更多了                    │
└──────────────────────────────────────────┘
```

### 1.2 轮播区域

- 使用 `ApplicationCarousel` 组件
- 数据：`getRecommendBanners()` API
- 最多 5 条
- padding: `2rem 2rem 0 2rem`

### 1.3 筛选/排序栏

```
┌──────────────────────────────────────────────┐
│ 全部  │  办公  │  教育  │  ...      排序: [默认 ▼] │
└──────────────────────────────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 容器 padding | 1rem = 16px |
| 背景 | `surfaceContainerLow` |
| 圆角 | 0.5rem = 8px |
| 阴影 | shadow-sm（轻微） |

分类 Tab：
| 属性 | 值 |
|------|-----|
| 类型 | 水平滚动标签组 |
| 激活态 | `primary` 文字 + 底部指示线 |
| 默认态 | `onSurfaceVariant` 文字 |
| 字号 | 14px |

排序下拉：
| 属性 | 值 |
|------|-----|
| 选项 | 默认、下载量最多、评分最高 |
| 类型 | `Select` → `DropdownButton` |
| 宽度 | auto |

### 1.4 无限滚动

使用 `usePaginatedList` + `useAutoLoadWhenNotScrollable`：
- 首屏：优先尝试 seed 缓存（构建期数据），否则骨架屏
- 触底加载：距底部 200px 触发
- 容器不可滚动时自动补页
- 保活页面重新可见时触发后台刷新

### 1.5 缓存策略

| 场景 | 缓存 key |
|------|---------|
| 推荐默认页 | `recommend_default` |
| 推荐+分类筛选 | `recommend_category_{categoryId}` |
| 推荐+排序 | `recommend_sort_{sortType}` |

Seed 数据：构建期预生成 `recommend_default` 的前 3 页。

```
┌─────────────────────────────────────────┐

├─────────────────────────────────────────┤

│ │           版本号  大小   开发者      │  │
│ │           评分 ★★★★☆ 4.5 (1.2k)    │  │
│ └────────────────────────────────────┘  │
│                                         │
│ ┌─── 截图轮播 ───────────────────────┐  │
│ │  [img1] [img2] [img3] [img4]       │  │
│ └────────────────────────────────────┘  │
│                                         │
│ ┌─── 应用介绍 ───────────────────────┐  │
│ │  描述文本...                        │  │
│ │  [展开/收起]                        │  │
│ └────────────────────────────────────┘  │
│                                         │
│ ┌─── 应用信息 ───────────────────────┐  │
│ │  分类:  办公                        │  │

│ │  更新:  2026-01-15                  │  │
│ └────────────────────────────────────┘  │
│                                         │
│ ┌─── 版本列表 ───────────────────────┐  │
│ │  v1.2.3 (当前)               [卸载] │  │
│ │  v1.2.2                      [卸载] │  │
│ └────────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

### 3.2 头部信息

| 元素 | 属性 |
|------|------|
| 图标 | 6rem × 6rem = 96×96px, 圆角 12px |
| 应用名 | 1.5rem = 24px, w600 |
| 版本号 | 0.875rem = 14px, secondary |
| 开发者 | 0.875rem = 14px, secondary |
| 评分 | Rate + 数字 + (评论数) |
| 操作按钮 | primary large，宽 120px |
| 次级操作 | 与主按钮同一行，位于主按钮右侧 |

头部 padding: `2rem`

业务约定：

- “创建桌面快捷方式”“卸载”不再折叠进右上角更多菜单，统一放在详情页头部，与主按钮同一行并排展示。
- 次级操作的显示条件必须来自本地安装态：仅当 `installedAppsProvider` 中存在当前 `appId` 的任一已安装实例时才渲染；未安装态直接隐藏，不能显示禁用按钮。
- 当应用已安装但存在更新时，次级操作仍然保持可见；它们的可见性只跟“是否已有本地安装实例”相关，不跟主按钮当前显示“打开”还是“更新”绑定。
- “应用信息”区采用三列信息栅格：短字段默认按三列同行排布，`包名 / 运行时 / 许可证 / 主页` 这类长字段必须独占整行，避免继续回退成纵向长表格。
- 详情页头部和“应用信息”中的体积字段必须走统一的人性化格式化工具，禁止直接渲染后端返回的裸字节串。

### 3.3 截图轮播

| 属性 | 值 |
|------|-----|
| 图片高度 | 200px |
| 圆角 | 8px |
| 间距 | 16px |
| 滚动 | 水平可滚动 |
| 空态 | 无截图不显示此区域 |

业务约定：

- 截图列表必须跟 `/app/getAppDetail` 的语言契约保持一致：请求体显式传 `lang`，由后端按语言精确过滤 `appScreenshotList` 后返回；Flutter 页面只消费返回结果，不在本地再做回退或二次筛选。
- 当前应用仅支持 `zh/en` 两套前台语言时，详情请求需要在仓储层把 locale 归一成后端约定值：`zh* -> zh_CN`，`en* -> en_US`。
- 截图区标题必须复用现有 i18n key `screenShots`，不能继续硬编码中文标题。

### 3.4 应用介绍

- 默认显示 3 行
- 超过 3 行显示 "展开" 按钮
- 展开后显示 "收起"

### 3.5 版本列表

版本列表始终展示该应用在当前仓库/架构下的可选版本，不是“仅展示本地已安装版本”：

| 列 | 内容 |
|----|------|
| 版本号 | `v{version}` |
| 标记 | 已安装版本显示“已安装” |
| 操作 | 未安装版本显示“安装”按钮，已安装版本不再重复提供安装入口 |

业务约定：

- 数据来源：`AppRepository.getVersions()` → `/visit/getSearchAppVersionList`
- 请求参数：必须携带 `appId + repoName + arch`，禁止只传 `appId` 依赖后端默认值；后端未显式传仓库时会回落到 `repo`，与社区版默认 `stable` 不一致
- 结果规范化：统一在 `AppRepository` 收敛；同版本存在多条 `module` 记录时优先保留 `module=binary`，随后按语义版本号倒序排序
- 安装态来源：行级“已安装”标记必须来自本地 `installedAppsProvider` 的已安装版本集合，不能使用详情接口返回的单一版本号冒充当前安装态
- 文件大小展示需对齐旧版 Rust 商店：空值或非法值显示 `--`，其余按 KB/MB/GB 做人性化格式化；版本历史与“应用信息”必须复用同一工具，不能继续直接显示裸字节串
- 失败处理：版本列表加载失败不能伪装成“暂无版本历史”；详情页主体保持可用，同时版本区展示轻量错误提示与重试入口

### 3.6 相关推荐

页面底部显示同分类的推荐应用（实现待定）。

## 状态驱动 UI 汇总

### 列表页通用状态流

```
InitialLoad (骨架屏)
    ↓ 有缓存?
    ├─ Yes → ShowCachedData → BackgroundRefresh → MergeUpdate
    └─ No  → FetchFirstPage
                 ↓ 成功?
                 ├─ Yes → ShowData → UserScroll → LoadMore
                 └─ No  → ShowError (重试按钮)
```

### ApplicationCard 状态流

```
           appInfo
              ↓
    ┌─────────┼───────────┐
    ↓         ↓           ↓
 未安装     已安装      安装中
  [安装]   [打开/更新]  [进度环]
```
