# flutter-linglong-store GitHub Release 更新日志条目生成 Prompt

你正在为 flutter-linglong-store 编写 GitHub Release 的用户可读更新日志条目。

## 当前任务范围

- 当前版本：{{RELEASE_VERSION}}
- 当前构建类型：{{BUILD_KIND}}
- 当前基线引用：{{BASELINE_REF}}
- 当前代码库根目录：{{WORKSPACE_ROOT}}
- 当前文档目录：{{DOCS_ROOT}}

说明：当前构建类型只允许为 `release` 或 `nightly`，当前工作运行在 GitHub Action 发布链路中。

## 分析要求

- 只基于输入上下文中的 release notes 范围、候选提交、变更文件和文档摘录总结。
- 目标是总结这次实际构建里用户可感知的变化，不要机械复述 commit 标题。
- 同一条用户功能链路必须合并成一条，例如协议解析、协议注册和安装入队应合并成“网页拉起客户端安装”。
- 文档、CI、workflow、打包、发布流程、工具链、AUR/UOS 发布、测试、重构、代码注释规范等维护性改动不能写进条目。
- 不要尝试修改文件，不要执行 git 提交，不要输出任何执行说明。

## 输出要求

- 使用简体中文。
- 说人话，内容清晰明了，让人一看就知道更新什么。
- 只能使用当前可访问上下文、代码库、文档和变更信息中已经明确支持的内容。
- 只输出 JSON，不要输出 Markdown。
- 顶层对象必须只有 `items` 字段。
- `items` 必须是字符串数组，最多 5 条。
- 每个数组元素必须是一句中文用户可读描述，不得包含编号，不得包含分类前缀。
- 只提及用户可感知的变化。
- 如果没有相关的用户可见变化，必须输出 `{"items":[]}`。
- 不要提到 AI、prompt，或上下文包的存在。
- 不要编造代码库、文档或当前变更里没有明确支持的功能或结论。

## 输出示例

```json
{"items":["支持从网页商店拉起客户端并加入安装队列。"]}
```
