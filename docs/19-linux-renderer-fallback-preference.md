# Linux 渲染模式自动回退与用户开关设计

## 背景

部分非 Intel / AMD CPU 设备在 Flutter Linux 默认 GPU 渲染路径下存在黑屏风险。为了降低首次启动失败率，需要在启动期自动为非白名单 CPU 切换到软件渲染，同时保留用户手动关闭软件渲染的选择权。

## 目标

- 非 Intel / AMD CPU 默认启用 `FLUTTER_LINUX_RENDERER=software`。
- 设置页提供软件渲染开关，但生效时机必须明确为“下次启动”。
- 如果用户通过环境变量显式设置渲染模式，则环境变量优先，设置页只能展示实际状态，不能与当前会话状态冲突。
- 用户关闭软件渲染时，在非白名单 CPU 上弹出风险提示，明确可能黑屏，并提供可复制的数据目录删除命令。
- CPU 白名单需要集中定义，后续新增厂商时改动面尽量小。

## 关键决策

### 1. 启动期真相源

新增一个 Linux runner 与 Dart 共享的启动配置文件，放在当前统一数据目录下。理由：

- `SharedPreferences` 初始化发生在 Dart `main()`，已经晚于 Linux runner 创建 Flutter 引擎。
- 启动配置文件可以在 C++ runner 创建 `FlView` 之前读取，也可以由设置页直接写入。
- 配置项很少，独立文件比反向推断 `SharedPreferences` 存储格式更稳定。

### 2. 渲染决策优先级

启动期按以下优先级决定实际渲染模式：

1. **用户环境变量**：若外部已设置 `FLUTTER_LINUX_RENDERER`，直接尊重，不再覆盖。
2. **用户设置**：若启动配置中存在显式偏好（software / hardware），按配置生效。
3. **CPU 自动回退**：若 CPU 不在白名单，则自动启用 software。
4. **默认硬件渲染**：其余情况保持默认。

### 3. 设置页交互

- 展示两个概念：
  - **当前渲染状态**：当前进程实际运行在软件 / 硬件渲染。
  - **下次启动偏好**：用户是否希望下次启动使用软件渲染。
- 外部环境变量生效时：
  - 开关状态跟随当前实际状态。
  - 开关禁用，并提示“当前由环境变量控制”。
- 用户自行切换开关时：
  - 保存到启动配置文件。
  - 通过对话框/通知明确提示“下次启动生效”。
  - 当前状态区继续如实显示“本次会话仍按当前渲染模式运行”。

### 4. 风险提示与恢复命令

当设备属于非白名单 CPU，且用户尝试关闭软件渲染时：

- 弹出风险确认对话框。
- 文案说明：可能因 GPU 渲染失败出现黑屏。
- 提供当前数据目录路径与可复制删除命令。
- 提醒用户提前保存命令，方便黑屏后在终端执行。

删除命令由统一数据目录解析逻辑生成，避免和历史目录、日志目录硬编码混淆。

## 数据与状态

### 启动配置文件

建议使用 INI / key-value 形式，仅保存最小必要信息：

- `preferred_mode=auto|software|hardware`

### 启动状态回传

Linux runner 在完成启动决策后，通过原生 channel 向 Dart 暴露：

- 当前实际模式
- 决策来源（env / user_preference / cpu_fallback / default）
- CPU vendor / model 摘要
- 是否命中 CPU 白名单
- 下次启动偏好

## CPU 白名单

Linux runner 内以集中常量表定义白名单，默认仅包含：

- `GenuineIntel`
- `AuthenticAMD`

要求：

- 单独放在一处常量定义。
- 匹配规则尽量宽松但可预测，优先基于 `/proc/cpuinfo` 的 `vendor_id`。
- 后续扩展时只需要新增常量项，不需要重写决策流程。

## 受影响模块

- `linux/runner/`：启动期渲染决策、状态桥接、CPU 白名单。
- `lib/core/storage/`：启动配置文件读写、数据目录命令生成。
- `lib/application/providers/`：设置状态恢复与渲染状态 provider。
- `lib/presentation/pages/setting/`：设置页开关、状态说明、风险弹窗。
- `lib/core/i18n/l10n/`：新增相关文案。
- `test/`：provider、setting page、native channel、路径/命令生成测试。

## 体验原则

- 不隐藏真实状态：当前会话与下次启动偏好必须区分清楚。
- 不和用户环境变量打架：外部显式配置优先，UI 只做解释，不自作主张覆盖。
- 不把用户逼进死角：关闭软件渲染前把回退命令直接摆出来，并支持复制。
- 不制造额外维护成本：CPU 白名单与命令生成都必须集中封装。
