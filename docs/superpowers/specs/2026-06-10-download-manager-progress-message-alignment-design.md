# 下载管理进度文案与提示对齐修复设计

**日期**：2026-06-10  
**状态**：用户已确认  
**范围**：下载管理弹窗当前任务主卡片

## 背景

下载管理弹窗的当前任务主卡片会同时在应用名下方和进度条上方展示同一条 `InstallTask.displayMessage`。当 ll-cli 输出 `Updating main:...` 这类阶段文案时，用户会看到重复内容，影响当前任务信息层级。

当前任务接近完成且持续时间较长时，会显示“如果进度看起来较慢，可能正在安装软件必备依赖，请再等等……”提示。提示左侧的黄色图标使用行顶部对齐，在中文 caption 行高中会显得偏上，和文字首行不够协调。

## 目标

- 当前任务主卡片只在进度条上方展示阶段文案，避免 `Updating...` 这类内容重复出现。
- 保留进度文案的完整 Tooltip、单行省略和进度百分比展示能力。
- 调整慢安装提示图标与文字首行的视觉对齐。
- 不改变安装队列、下载速度、复制日志、取消任务、历史记录等业务行为。

## 非目标

- 不修改 `InstallTask` 状态机或进度解析逻辑。
- 不新增 `ll-cli` 调用或数据源。
- 不改下载管理弹窗整体工作面板结构。
- 不改变等待队列和历史记录紧凑行的副标题展示。

## 方案

采用最小展示层修复：

1. 在当前任务主卡片的标题区，`featured=true` 时不再把 `displayMessage` 拼进应用名下方的副标题。标题区只承担应用身份信息展示，阶段文案继续由进度区展示。
2. 等待队列和历史记录仍使用现有 `_buildSubtitle()`，避免影响失败、已完成、排队等紧凑任务行的信息密度。
3. 慢安装提示行保持现有文案和颜色，只调整图标布局，使 14px 图标相对 caption 文本首行居中，而不是贴着整行顶部。

## 测试策略

- 在 `test/widget/presentation/widgets/download_manager_dialog_test.dart` 增加当前任务进度文案去重测试：构造安装中的当前任务，断言相同阶段文案只出现一次。
- 复用现有慢安装提示测试，补充图标与提示文案处于同一个提示行的结构断言。
- 运行定向 widget 测试，确认下载管理弹窗既有行为不回退。
- 运行 `flutter analyze`，确认没有新增静态分析问题。

## 验收标准

- 截图中的 `Updating main:...` 只保留进度条上方一处。
- 慢安装提示图标与文字首行视觉对齐。
- 下载管理弹窗复制日志、取消任务、进度百分比和底部速度展示行为不变。
- 定向 widget 测试通过，静态分析不新增 error 或 warning。
